# 02. 페르소나

## 주 페르소나: 지영 (28세, 직장인)

### 프로필
- 사무직 직장인, 하루 대부분 앉아서 근무
- 다이어트 목적으로 칼로리 관리 시작
- 과거에 다른 칼로리 앱을 써봤지만 입력이 번거로워서 포기한 경험

### 목표
- 점심·저녁 먹을 때마다 바로 기록해서 하루 섭취량 파악
- "오늘 얼마나 더 먹을 수 있지?"를 즉시 확인
- 남은 칼로리 안에서 뭘 먹으면 좋을지 추천받기

### 불만 (기존 도구)
- 음식 검색이 느리고 결과가 영어 위주
- 영양소 입력 항목이 너무 많아 귀찮음
- 앱이 무겁고 회원가입이 필수

### 행동 패턴
- 매 끼니 직후 폰으로 기록 (하루 2~3회)
- 빠른 입력을 선호, 30초 안에 끝내고 싶음
- 저녁에 간식 먹기 전 남은 칼로리 확인

---

## 대조 페르소나: 상훈 (52세, 회사원)

### 프로필
- 건강검진에서 고지혈증 경고를 받고 식단 관리 시작
- 칼로리 관리 경험 전무, 앱 사용에 익숙하지 않음
- 아내가 "이거 써봐"라고 권유해서 시작

### 목표
- 의사가 권한 일일 칼로리 범위 안에서 먹기
- 어떤 음식이 좋고 나쁜지 가이드를 받고 싶음
- 복잡한 건 싫고 숫자 하나만 보고 싶음

### 불만 (기존 도구)
- 뭘 검색해야 하는지 모르겠음
- 화면에 정보가 너무 많으면 포기
- 영어 음식명은 알아볼 수 없음

### 행동 패턴
- 하루 1~2회 기록 (점심, 저녁)
- 수동 입력보다 검색 선호 (뭘 입력해야 할지 모르니까)
- 추천 음식 목록에 크게 의존

---

## 긴장 축 비교

| 축 | 지영 (주) | 상훈 (대조) |
|---|---|---|
| 동기 | 다이어트 (자발적) | 건강 경고 (외부 압력) |
| 숙련도 | 앱 익숙, 칼로리 개념 있음 | 앱 미숙, 칼로리 개념 약함 |
| 입력 선호 | 수동 입력도 OK (빠르면) | 검색 의존 (뭘 쓸지 모름) |
| 핵심 화면 | 남은 칼로리 숫자 | 추천 음식 목록 |
| 이탈 위험 | 입력이 귀찮으면 포기 | 화면이 복잡하면 포기 |

## 설계 긴장점

1. **입력 속도 vs 입력 가이드**: 지영은 빠르게 치고 싶고, 상훈은 뭘 쳐야 할지 안내가 필요 → 검색 + 수동 입력 탭 분리로 해결
2. **정보 밀도 vs 단순함**: 지영은 숫자를 보고 판단하고, 상훈은 "이거 먹어도 돼?"만 알고 싶음 → 대시보드는 큰 숫자 중심, 추천은 별도 섹션
3. **자율성 vs 가이드**: 지영은 직접 고르고, 상훈은 추천에 의존 → 추천을 눈에 띄게 배치하되 강제하지 않음
