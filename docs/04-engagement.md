# 04. 차별화/성취감

## 선택된 기능

### E1. 한 줄 요약 코멘트
- 현재 섭취 상태를 자연어로 요약: "점심까지 잘하고 있어요, 저녁은 600kcal 이내로!"
- 남은 칼로리 구간별 메시지 분기:
  - 70% 이상 남음: "여유 있어요, 든든하게 드세요!"
  - 30~70% 남음: "잘하고 있어요, 저녁은 가볍게!"
  - 10~30% 남음: "거의 다 왔어요, 간식은 참아볼까요?"
  - 초과: "오늘은 좀 넘었어요. 내일 가볍게 시작해봐요!"
- 대시보드 상단에 항상 표시

### E2. 빠른 추가 버튼
- 자주 먹는 기본 음식을 원터치로 추가: "밥 한 공기 +300", "계란 1개 +80", "우유 한 잔 +130"
- 기록 화면 상단에 가로 스크롤 칩 버튼으로 배치
- 3탭(검색→선택→추가)을 1탭으로 단축 — 설계 원칙 P2 강화

### E3. 하루 마감 요약
- 하루가 끝날 때(자정 리셋 시점) 요약 카드 표시:
  - "오늘 1,520kcal 섭취, 목표 대비 92% 달성"
  - 먹은 음식 개수, 가장 칼로리 높았던 음식
- 다음 날 앱 열면 어제 요약을 잠깐 보여주고 새 하루 시작

### E4. 좋은 음식/나쁜 음식 태그
- 음식 DB에 건강 태그 부여:
  - 🟢 좋음: 샐러드, 닭가슴살, 잡곡밥, 과일 등
  - 🔴 주의: 치킨, 피자, 라면, 탄산음료 등
  - ⚪ 보통: 그 외
- 검색 결과, 추천 목록, 오늘 기록에 태그 색상 표시
- 추천 시 좋은 음식 우선 노출

## 시나리오 영향

| 기능 | 영향받는 시나리오 | 효과 |
|---|---|---|
| E1. 한 줄 요약 | S2, S4, S7 | 숫자 대신 자연어로 상태 파악 — 상훈도 직관적으로 이해 |
| E2. 빠른 추가 | S3, S6 | 반복 음식 기록 시간 1/3로 단축 — 지영의 이탈 방지 |
| E3. 하루 마감 | S4 | 하루 돌아보기로 다음 날 동기 부여 |
| E4. 건강 태그 | S2, S7, S8 | 뭘 먹어야 좋은지 한눈에 — 상훈의 핵심 니즈 충족 |

## 의도적으로 넣지 않는 것

- **스트릭/연속 기록**: 반나절 MVP에서는 오늘만 다루므로 히스토리 필요한 기능 제외
- **프로그레스 링 애니메이션**: 시각적 효과보다 기능 완성도 우선
- **초과 시 다음 끼니 가이드**: 한 줄 요약(E1)이 이미 유사 역할 수행
