# 01. 요구사항

## 프로젝트 개요

- **한 줄 정의**: 일일 칼로리 목표 대비 섭취량을 추적하고, 남은 칼로리에 따라 음식을 추천하는 앱
- **목적**: 개인 사용
- **주 사용 디바이스**: 반응형 (모바일 + 데스크톱)
- **기술 스택**: Next.js
- **데이터 저장**: localStorage

## 기능 요구사항

### 데이터 모델

- **사용자 프로필**: 키(cm), 몸무게(kg), 나이, 성별, 활동량 → TDEE 자동 계산
- **음식 기록**: 음식명, 칼로리(kcal), 기록 시간, 입력 방식(검색/수동)
- **일일 요약**: 목표 칼로리, 섭취 칼로리, 남은 칼로리

### 핵심 기능

1. **TDEE 계산기**: 키/몸무게/나이/성별/활동량 입력 → 일일 권장 칼로리 자동 산출
2. **음식 입력 (2가지 방식)**:
   - 검색 입력: 음식 DB에서 검색하여 선택
   - 수동 입력: 음식명 + 칼로리 직접 타이핑
3. **칼로리 대시보드**: 목표 대비 섭취량 시각화, 남은 칼로리 표시
4. **상황별 음식 추천**: 남은 칼로리에 따라 먹을 수 있는 좋은 음식 제안
5. **오늘 기록 관리**: 오늘 먹은 음식 목록 표시, 삭제 가능, 하루 단위 리셋

### 영양소 추적 범위

- 칼로리(kcal)만 추적 (3대 영양소는 범위 제외)

## 비기능 요구사항

- 반응형 레이아웃 (모바일 우선)
- 빠른 음식 검색 응답
- 한글 음식명 지원
- 자정 기준 일일 리셋

## 범위 제외 (Out of Scope)

- 회원가입/로그인
- 서버/DB 연동
- 주간/월간 히스토리 및 트렌드
- 3대 영양소(탄/단/지) 추적
- 운동 소모 칼로리
- 사진으로 음식 인식
- 바코드 스캔

## 완료 기준 (Definition of Done)

- [ ] 사용자 프로필 입력 시 TDEE가 정확히 계산됨
- [ ] 음식을 검색/수동 두 가지 방식으로 추가할 수 있음
- [ ] 대시보드에서 목표/섭취/남은 칼로리가 실시간 반영됨
- [ ] 남은 칼로리에 맞는 음식이 추천됨
- [ ] 모바일/데스크톱 모두 정상 표시됨
- [ ] localStorage에 오늘 데이터가 저장/리셋됨
