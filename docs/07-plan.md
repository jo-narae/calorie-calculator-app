# 07. 구현 계획

## 시간 예산: 반나절 (2~3시간)

## v2 미룸 항목

| 기능 | 이유 |
|---|---|
| E3. 하루 마감 요약 | 히스토리 저장 로직 필요, MVP 핵심 아님 |
| E4. 좋은/나쁜 음식 태그 | 음식 DB 태그 데이터 정제 후 적용 |
| 검색 실패 폴백 (카테고리 평균 칼로리) | 기본 검색 + 수동 입력으로 커버 가능 |

## 마일스톤

| M | 목표 | 끝나면 보이는 것 | 예상 시간 |
|---|---|---|---|
| M1 | 프로젝트 셋업 + 기본 구조 | Next.js + Tailwind 실행, 빈 페이지 | 15분 |
| M2 | 데이터 레이어 | 타입, Context, localStorage, TDEE 계산, 가상 음식 DB | 20분 |
| M3 | 프로필 입력 + 온보딩 | 키/몸무게/나이/성별/활동량 입력 → TDEE 표시 | 20분 |
| M4 | 대시보드 UI | 원형 링, 통계 카드, 한 줄 코멘트 — 실시간 반영 | 25분 |
| M5 | 음식 입력 (검색 + 수동 + 빠른 추가) | 모달에서 검색/수동 입력, 칩으로 빠른 추가 | 30분 |
| M6 | 오늘 기록 + 추천 + 마무리 | 먹은 음식 리스트, 삭제, 추천 섹션, 일일 리셋, 반응형 | 30분 |

## M1. 프로젝트 셋업

**할 일:**
- Next.js (App Router) + TypeScript 프로젝트 생성
- Tailwind CSS 설치 및 설정
- 디자인 토큰을 tailwind.config.ts에 반영 (컬러, 그라데이션)
- globals.css에 커스텀 CSS 변수 추가
- 루트 레이아웃 (layout.tsx) 기본 구성
- 빈 page.tsx 확인

**검증:**
- `npm run dev` → 브라우저에서 빈 페이지 정상 표시
- Tailwind 클래스 적용 확인

---

## M2. 데이터 레이어

**할 일:**
- `types/index.ts` — 타입 정의 (UserProfile, FoodItem, FoodRecord, DailyData)
- `data/foods.json` — 가상 한글 음식 DB (100~150개)
- `lib/tdee.ts` — TDEE 계산 함수 (Mifflin-St Jeor)
- `lib/storage.ts` — localStorage CRUD + 날짜 비교 리셋
- `lib/search.ts` — 문자열 매칭 검색
- `lib/recommend.ts` — 남은 칼로리 기반 추천
- `context/CalorieContext.tsx` — 전역 상태 Provider

**검증:**
- TDEE 계산: 170cm/70kg/30세/남성/보통활동 → 약 2,300kcal 근처
- 음식 검색: "비빔" 입력 → 비빔밥 등 결과 반환
- localStorage 저장/로드 동작 확인

---

## M3. 프로필 입력

**할 일:**
- `ProfileSetup.tsx` — 키/몸무게/나이/성별/활동량 폼
- 활동량 기본값 "보통" 선택
- 입력 완료 → TDEE 계산 → Context에 저장 → localStorage 저장
- 프로필 있으면 대시보드, 없으면 프로필 입력 화면 분기

**검증:**
- 프로필 입력 후 대시보드로 전환
- 새로고침 후에도 프로필 유지
- 30초 이내 입력 가능 (P1)

---

## M4. 대시보드 UI

**할 일:**
- `CalorieRing.tsx` — SVG 원형 링 차트 (그린→오렌지 그라데이션)
- `CommentCard.tsx` — 남은 칼로리 구간별 이모지 + 메시지
- `StatCards.tsx` — 목표/섭취 2열 카드
- page.tsx에 조합 — 명랑 헬시 디자인 적용
- 칼로리 초과 시 링 색상 빨간색 전환 + 코멘트 변경

**검증:**
- 원형 링이 섭취량에 따라 애니메이션
- 코멘트가 구간별로 변경됨
- 초과 시 빨간색 경고

---

## M5. 음식 입력

**할 일:**
- `AddFoodModal.tsx` — 하단 시트/모달
  - 검색 탭: 입력 → 실시간 필터링 → 선택 → 추가
  - 수동 탭: 음식명 + 칼로리 직접 입력 → 추가
- `QuickAddChips.tsx` — 자주 먹는 음식 원터치 추가
- FAB 버튼 → 모달 열기

**검증:**
- 검색 "비빔" → 결과 표시 → 선택 → 대시보드 반영
- 수동 입력 → 대시보드 반영
- 칩 탭 → 즉시 추가 → 대시보드 반영
- 10초 이내 기록 가능 (P2)

---

## M6. 오늘 기록 + 추천 + 마무리

**할 일:**
- `FoodList.tsx` — 오늘 먹은 음식 리스트 + 삭제 버튼
- `RecommendSection.tsx` — 남은 칼로리 이내 음식 추천
- 일일 리셋: 앱 로드 시 날짜 비교 → 다른 날이면 데이터 초기화
- 반응형: 모바일(~480px) / 태블릿 / 데스크톱(480px 중앙 정렬)
- 전체 플로우 점검

**검증:**
- 음식 삭제 → 대시보드 실시간 반영
- 추천 음식이 남은 칼로리 이내로만 표시
- 모바일/데스크톱 레이아웃 정상
- localStorage 영속성 확인
